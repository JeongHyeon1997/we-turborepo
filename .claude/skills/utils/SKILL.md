# Utils Skill

인라인 유틸리티 로직을 `utils/`로 추출하거나 기존 함수로 교체한다.

## 규칙

- 2곳 이상에서 쓰이거나 쓰일 가능성이 높은 순수 함수 → `utils/`에 추출
- 도메인 로직(API 호출, store 의존)이 없는 순수 변환/계산 함수만 대상
- 파일 분리 기준: 역할별 (`format.ts`, `validation.ts`, `storage.ts` 등)
- 기존 `utils/`에 동일 역할 함수가 있으면 신규 생성 대신 기존 파일에 추가

## 실행 순서

1. 대상 파일에서 인라인 유틸리티 로직 파악
2. 기존 `utils/` 함수와 대조 (이미 있으면 교체, 없으면 추가)
3. 계획 출력 후 확인 요청
4. 승인 후 utils 파일 업데이트 + 컴포넌트에서 import 교체

## 출력 형식

```
[발견된 인라인 유틸 로직]
- {설명} (line {번호})
  처리: 기존 {함수명} 으로 교체 / utils/{파일명}.ts 에 {함수명} 추가

진행할까요?
```
